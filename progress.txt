## Codebase Patterns
- TypeScript project: use `npm run typecheck` (tsc --noEmit) before committing
- Run scripts with `npx tsx scripts/<name>.ts`
- IronClaw CLI: `ironclaw status` for health, `ironclaw config list` for settings, `ironclaw doctor` for diagnostics
- IronClaw stores secrets in macOS keychain (`secrets_master_key_source: keychain`), not in env vars
- IronClaw config is in `~/.ironclaw/.env` (database-backed, use `ironclaw config set` to modify)
- PostgreSQL for IronClaw runs via Homebrew on /tmp:5432 (not Docker)
- IronClaw MCP servers managed via `ironclaw mcp list` / `ironclaw mcp add`
- Always update AGENTS.md Environment Details table when infrastructure changes
- Docker Desktop settings at `~/Library/Group Containers/group.com.docker/settings.json`
- pmset changes require sudo ‚Äî scripts that need it should document the sudo requirement clearly
- Docker `--restart unless-stopped` is a convention enforced in all `docker run` commands (not automated tooling)
- Neo4j 5.26.0 pinned (NOT latest 5.26.x) ‚Äî GDS plugin has no compatible version for 5.26.21+
- Docker Desktop memory must be ‚â•16GB for Neo4j 8GB heap ‚Äî settings.json `memoryMiB: 16384`
- Neo4j auth: `neo4j` / `lingelpedia2026` ‚Äî container name: `linglepedia`
- Neo4j data persists at `$HOME/neo4j/data`, logs at `$HOME/neo4j/logs`
- Use `cypher-shell -u neo4j -p lingelpedia2026` inside the container for Cypher queries
- Pipe Cypher queries via stdin (`docker exec -i ... cypher-shell` with `input:`) ‚Äî backticks in Cypher get mangled by shell when passed as args
- All Cypher DDL uses `IF NOT EXISTS` for idempotency ‚Äî schema can be re-applied safely
- Reusable Cypher files live in `cypher/` directory
- MCP servers for IronClaw run as HTTP services ‚Äî register with `ironclaw mcp add <name> <url>`
- mcp-neo4j-cypher uses SSE (text/event-stream) responses ‚Äî parse with `data: ` prefix matching
- Use launchd LaunchAgents (~/Library/LaunchAgents/) for persistent MCP server processes with KeepAlive + RunAtLoad
- uvx caches Python packages ‚Äî first run downloads, subsequent runs are fast
- IronClaw `mcp test <name>` verifies connectivity and lists available tools
- MCP SDK StreamableHTTPServerTransport stateless mode: create new server + transport per request, not shared
- Vault directory is `Linglepedia` (not `Lingelpedia`) at `~/Library/Mobile Documents/com~apple~CloudDocs/Linglepedia`
- Node.js `fs.watch({ recursive: true })` uses FSEvents natively on macOS ‚Äî no extra library needed
- MCP servers on different ports: Neo4j=8765, vault=8766. Next server should use 8767+

## Codebase Patterns
- TypeScript project: use `npm run typecheck` (tsc --noEmit) before committing
- Run scripts with `npx tsx scripts/<name>.ts`
- IronClaw CLI: `ironclaw status` for health, `ironclaw config list` for settings, `ironclaw doctor` for diagnostics
- IronClaw stores secrets in macOS keychain (`secrets_master_key_source: keychain`), not in env vars
- IronClaw config is in `~/.ironclaw/.env` (database-backed, use `ironclaw config set` to modify)
- PostgreSQL for IronClaw runs via Homebrew on /tmp:5432 (not Docker)
- IronClaw MCP servers managed via `ironclaw mcp list` / `ironclaw mcp add`
- Always update AGENTS.md Environment Details table when infrastructure changes
- Docker Desktop settings at `~/Library/Group Containers/group.com.docker/settings.json`
- pmset changes require sudo тАФ scripts that need it should document the sudo requirement clearly
- Docker `--restart unless-stopped` is a convention enforced in all `docker run` commands (not automated tooling)
- Neo4j 5.26.0 pinned (NOT latest 5.26.x) тАФ GDS plugin has no compatible version for 5.26.21+
- Docker Desktop memory must be тЙе16GB for Neo4j 8GB heap тАФ settings.json `memoryMiB: 16384`
- Neo4j auth: `neo4j` / `lingelpedia2026` тАФ container name: `linglepedia`
- Neo4j data persists at `$HOME/neo4j/data`, logs at `$HOME/neo4j/logs`
- Use `cypher-shell -u neo4j -p lingelpedia2026` inside the container for Cypher queries
- Pipe Cypher queries via stdin (`docker exec -i ... cypher-shell` with `input:`) тАФ backticks in Cypher get mangled by shell when passed as args
- All Cypher DDL uses `IF NOT EXISTS` for idempotency тАФ schema can be re-applied safely
- Reusable Cypher files live in `cypher/` directory
- MCP servers for IronClaw run as HTTP services тАФ register with `ironclaw mcp add <name> <url>`
- mcp-neo4j-cypher uses SSE (text/event-stream) responses тАФ parse with `data: ` prefix matching
- Use launchd LaunchAgents (~/Library/LaunchAgents/) for persistent MCP server processes with KeepAlive + RunAtLoad
- uvx caches Python packages тАФ first run downloads, subsequent runs are fast
- IronClaw `mcp test <name>` verifies connectivity and lists available tools
- MCP SDK StreamableHTTPServerTransport stateless mode: create new server + transport per request, not shared
- Vault directory is `Linglepedia` (not `Lingelpedia`) at `~/Library/Mobile Documents/com~apple~CloudDocs/Linglepedia`
- Node.js `fs.watch({ recursive: true })` uses FSEvents natively on macOS тАФ no extra library needed
- MCP servers on different ports: Neo4j=8765, vault=8766. Next server should use 8767+
- xAI has NO embedding models (confirmed 2026-03-02) тАФ use Ollama nomic-embed-text (768 dims) as local fallback
- Ollama runs at http://localhost:11434 with OpenAI-compatible API at /v1/embeddings
- Embedding pipeline module: `src/embedding.ts` тАФ import { generateEmbedding, storeClaimEmbedding, findSimilarClaims }
- Neo4j vector index dimensions MUST match the embedding model (currently 768 for nomic-embed-text)
- To change embedding dimensions: drop + recreate the vector index AND update cypher/lingelpedia-schema.cypher
- IronClaw identity system uses workspace memory files (IDENTITY.md, SOUL.md, AGENTS.md, USER.md, MEMORY.md), NOT YAML files
- `ironclaw memory write <path>` writes to workspace, `ironclaw memory read <path>` reads, `ironclaw memory tree` shows structure
- `ironclaw memory write` accepts content via stdin (pipe) or as second arg тАФ use stdin for multi-line content

- MCP server OAuth tokens stored in macOS Keychain (service: `vega-<service-name>`, account: `refresh_token`)

- `cargo-component` required for building IronClaw WASM tools тАФ install with `cargo install cargo-component`

  - Dual approach: custom TypeScript MCP server (HTTP, port 8767) AND IronClaw native WASM tool (google-calendar-tool)
  - Custom MCP server handles OAuth independently (stores refresh token in macOS Keychain), while WASM tool uses IronClaw's internal auth

  - Token refresh handled automatically тАФ access tokens cached with 60s buffer before expiry
- **Learnings for future iterations:**
  - Google OAuth requires interactive browser flow тАФ no way to fully automate from non-interactive sessions
  - The `ironclaw tool auth` command also requires browser interaction (opens Google consent page)
  - `ironclaw tool install` needs `cargo-component` (Rust) + `wasm32-wasip2` target тАФ install these first
  - Google OAuth credentials are shared at `~/Library/Application Support/gogcli/credentials.json` (from `gog` CLI)
  - `gog` CLI (v0.11.0) at /opt/homebrew/bin/gog has existing Google tokens but they're encrypted in a file keyring requiring GOG_KEYRING_PASSWORD
  - For MCP servers that need OAuth, use the pattern: local HTTP callback server тЖТ capture auth code тЖТ exchange for tokens тЖТ store refresh_token in Keychain
  - Keychain entries: `security add-generic-password -s <service> -a <account> -w <value> -U` (U = update if exists)
  - The OAuth callback port (8769) is separate from the MCP server port (8767) тАФ callback only runs during setup
  - IronClaw WASM tools and MCP servers are complementary тАФ WASM is native integration, MCP is HTTP protocol. Both can coexist.

- Mock call counts accumulate across tests — use `mockFn.mockClear()` before asserting exact call counts
- Neo4j vector search: `CALL db.index.vector.queryNodes('index_name', $topK, $embedding)` returns `node, score`

- Neo4j integer results need `.toNumber()` conversion (similar to PostgreSQL BIGINT → Number())

- Mocking global fetch in vitest: `vi.stubGlobal("fetch", mockFn)` + `vi.unstubAllGlobals()` in cleanup
- Ollama API: `/api/tags` returns `{ models: [{ name: "model:tag" }] }` — use for availability checks
- Frontier-only tasks: claim_decomposition, contradiction_detection, aqm_query, aqm_synthesis, initial_seeding, complex_reasoning
- Local-eligible tasks: formatting, embedding_preprocessing, routine_update, pre_reflection
- Frontmatter regex: `/^---\r?\n([\s\S]*?)\r?\n---\r?\n?([\s\S]*)$/` — captures YAML block and body separately
- VaultConnector write restriction: only `_agent_insights/` directory is writable — enforce with path.resolve + startsWith check
- Mock icloud-sync in vault tests: `vi.mock("../telemetry/icloud-sync.js")` to avoid needing `brctl` binary
- fs.watch with `{ recursive: true }` works on macOS for recursive directory monitoring
- File watcher tests are timing-sensitive — use delays and filter for specific events to avoid catching events from prior tests
- Migration templates live in `src/knowledge/migration/` — parser auto-detects type from frontmatter `type` field or `truth_tier` presence for claims
- Zod `z.literal()` enforces discriminator values; `errorMap` on `z.enum()` customizes error messages
- Neo4j MERGE idempotency: match on `{name: $name, entity_type: $entityType}` — use ON CREATE SET for initial props, ON MATCH SET for updates
- Claim extraction from body: split on paragraphs (double newline), bullet points (`- `, `* `, `+ `), and numbered lists — skip headings
- Neo4j `created_at = updated_at` comparison in RETURN clause detects whether MERGE did CREATE or MATCH
- Source nodes use `source-obsidian-{sanitized-path}` as ID for MERGE idempotency
- Shared `extractClaimsFromBody()` and `MigrationStats` type exported from entity-mapper.ts — reuse in all mappers
- ClaimDecomposer uses injectable `llmCall` function for testing — avoids needing live LLM in tests
- Domain classifications: gix, we, personal_finance, health, family, legal, general
- Truth tier assignment: family_direct (Mike/Lindsay, score ≥ 0.95), multi_source_verified, single_source, agent_inferred
- Decomposition deduplication: embedding cosine similarity > 0.95 = potential duplicate, skip creation
- OpenAI-compatible chat completions via fetch — works with xAI/Grok endpoint at `${baseURL}/chat/completions`
- Telemetry outcome type is `"success" | "failure" | "partial" | "skipped"` — must match emitter's Zod enum exactly
- Mock Neo4j session pattern: `{ session: () => ({ run: vi.fn().mockResolvedValue({ records: [...] }), close: vi.fn() }) }`
- Floating-point scoring: use `toBeCloseTo()` instead of `toBe()` for computed scores with addition/subtraction
- Cross-domain queries: `MATCH (c:Claim)-[:ABOUT]->(e:Entity {name: $name}) WHERE c.domain <> $domain` finds entity bridges

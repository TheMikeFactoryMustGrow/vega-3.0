## Codebase Patterns
- TypeScript project uses ESM (`"type": "module"`) with `moduleResolution: "bundler"` in tsconfig
- Use Zod for runtime schema validation of all data structures (events, frontmatter, etc.)
- Telemetry modules live in `src/telemetry/` — each concern in its own file (types.ts, emitter.ts, rotation.ts)
- Non-blocking pattern: wrap all I/O in try/catch, return null on failure, log to stderr
- File paths use `{YYYY-MM-DD}.jsonl` naming convention (no "events-" prefix)
- Tests use vitest with temp directories (`mkdtemp`) for filesystem isolation
- All exports go through `index.ts` barrel files
- Base telemetry path: `~/vega-telemetry/` with subdirs `events/`, `archive/`
- PostgreSQL `BIGINT` columns return strings in node-pg — always use `Number()` coercion in parse functions
- Use `pg` (node-postgres) with default import: `import pg from "pg"; const { Pool } = pg;` (ESM compat)
- Tier 2 tables use `ON CONFLICT DO UPDATE` for idempotent upserts — conflict keys are the UNIQUE constraints
- Database connection: `VEGA_PG_URL` env var for prod, `VEGA_TEST_PG_URL` for tests (default: `postgres://localhost/vega_test`)
- Tier 2 types in `tier2-types.ts`, migration DDL in `database.ts`, CRUD in `tier2-repository.ts`
